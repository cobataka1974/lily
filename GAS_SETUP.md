# GASセットアップ手順書

みつこさんの会話メモリをGoogle Sheetsに保存するためのセットアップ手順です。  
GASを使うと、ローカルの `data/sessions.json` の代わりにクラウドのGoogle Sheetsにデータが保存され、サーバー不要でVercelなどから利用できます。

---

## 1. Google Sheetsで新しいスプレッドシートを作成する

1. ブラウザで [https://sheets.google.com](https://sheets.google.com) を開く
2. 左上の「＋ 新しいスプレッドシート」をクリック
3. タイトルを「リリー セッションメモ」などに変更（任意）
4. **URLバーのIDをコピーしておく**  
   例：`https://docs.google.com/spreadsheets/d/`**`1aBcDeFgHiJkLmNo`**`/edit`  
   → ★ここの太字部分が `SHEET_ID` です。後で使います

---

## 2. Google Apps Scriptを開く

1. スプレッドシートのメニューから **「拡張機能」** → **「Apps Script」** をクリック
   - ★ここで新しいタブが開き、スクリプトエディタが表示されます
2. 左上のプロジェクト名「無題のプロジェクト」をクリックして **「リリーバックエンド」** に変更（任意）

---

## 3. gas/Code.gs の内容をコピーする

1. スクリプトエディタの左側「ファイル」欄にある `コード.gs` をクリック
2. エディタ内のコードを全選択（Ctrl+A / Cmd+A）して削除
3. このリポジトリの `gas/Code.gs` の内容を全てコピー＆ペースト
4. **「保存」ボタン（フロッピーアイコン）** をクリック、またはCtrl+S

---

## 4. スクリプトプロパティ3つを設定する

APIキーなどの機密情報はスクリプトプロパティに保存します。

1. スクリプトエディタ左サイドバーの **「プロジェクトの設定」（歯車アイコン）** をクリック
2. 画面を下にスクロールして **「スクリプト プロパティ」** セクションを表示
3. **「スクリプト プロパティを追加」** をクリックして以下の3つを設定：

| プロパティ名 | 値 |
|---|---|
| `ANTHROPIC_API_KEY` | AnthropicのAPIキー（sk-ant-...） |
| `ELEVENLABS_API_KEY` | ElevenLabsのAPIキー |
| `SHEET_ID` | 手順1でコピーしたスプレッドシートのID |

- ★ここで「スクリプト プロパティを追加」を3回押して1つずつ入力してください
- ★入力後、右下の **「スクリプト プロパティを保存」** を必ずクリックしてください

---

## 5. WebアプリとしてデプロイしてURLを取得する

1. スクリプトエディタ右上の **「デプロイ」** ボタンをクリック
2. **「新しいデプロイ」** を選択
3. ★ここで「デプロイの種類を選択してください」画面が出たら、歯車アイコンをクリックして **「ウェブアプリ」** を選んでください
4. 以下のように設定：
   - **説明**：`リリーバックエンド v1`（任意）
   - **次のユーザーとして実行**：`自分`
   - **アクセスできるユーザー**：`全員`（匿名含む）
5. **「デプロイ」** ボタンをクリック
6. ★ここで「アクセスを承認」画面が出た場合：
   - 「アクセスを承認」→ Googleアカウントを選択 → 「詳細」→「リリーバックエンド（安全ではないページ）に移動」→「許可」
7. デプロイ完了後、**「ウェブ アプリ URL」** が表示されます  
   例：`https://script.google.com/macros/s/AKfycby.../exec`
8. ★このURLをコピーしてください

---

## 6. .env.local の GAS_WEB_APP_URL に設定する

プロジェクトルートに `.env.local` ファイルを作成（または編集）して、コピーしたURLを貼り付けます：

```
# .env.local

ANTHROPIC_API_KEY=sk-ant-xxxxxx
ELEVENLABS_API_KEY=xxxxxx

# GAS Web App URL（手順5でコピーしたURL）
GAS_WEB_APP_URL=https://script.google.com/macros/s/AKfycby.../exec
```

> `.env.local` は `.gitignore` に含まれているためGitにはコミットされません。

---

## 7. 動作確認

### ローカルサーバーで確認する

```bash
node server.js
```

ブラウザで `http://localhost:3000/` を開き、以下を確認：

1. **起動時に「記憶を読み込み中…」** → `✅ SDK読み込み完了` と表示されればメモリ取得成功
2. **会話後に「会話を分析中…」** → `✅ 分析が完了しました` と表示されれば保存成功

### GAS を直接テストする

GASスクリプトエディタで関数を手動実行して確認できます：

1. エディタ上部の関数選択ドロップダウンで **`doGet`** を選択
2. **「実行」** ボタンをクリック
3. ★ここで「実行ログ」に `{"status":"ok","sessions":[],...}` などが出れば正常です

### スプレッドシートを確認する

会話終了後にGoogle Sheetsを開き、**「セッション」** シートに行が追加されていれば完了です。

---

## トラブルシューティング

| 症状 | 対処 |
|---|---|
| 「アクセスできません」エラー | 手順5の「アクセスできるユーザー」が「全員」になっているか確認 |
| `SHEET_ID が設定されていません` | 手順4のスクリプトプロパティを再確認 |
| 分析が保存されない | GASのデプロイURLが最新か確認（変更後は再デプロイが必要） |
| `doGet` が `{status:"error"}` を返す | Apps Scriptの「実行ログ」でエラー詳細を確認 |

---

*このセットアップにより、会話の分析結果がGoogle Sheetsにクラウド保存され、次回の会話に活かされます。*
